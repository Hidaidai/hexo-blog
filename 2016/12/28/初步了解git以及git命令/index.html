初步了解git以及git命令

Dec 28, 2016 | Linux学习 | Hits

  昨天基本做好了个人博客，因为暂时博客只面向国内（国外谁看我啊）用Coding比GitHub会快的多，所以我选择把个人博客托管在coding上。
  好吧！让我们先了解下什么是git、coding、github；然后熟悉下基本的git命令。

1.什么是git？

  git是目前世界上最先进的分布式版本控制系统，没有之一。
  First是分布式。分布式系统大家总知道吧，最著名的分布式系统就是万维网了。在万维网上，每个人看到的都是同一个网页，但其实都是一个网站。这里git对做好的项目进行管理，世界各地的人都可以看到，然后下载下来，随意操作。
  Then版本控制是关键。比如：今天我做了一个项目，基本功能都实现了，休息下明天继续；然后明天添加一个功能时，突然报错了，把新功能注释掉还是有错，但错在哪始终找不到，超纠结。但如果有了版本控制，每次结束任务，都commit下项目，设置为一个版本并加上一些注释，等下次继续项目时，即使有问题了，通过对比两次版本可以清楚找到有哪些变化。此外还有很多好处，比如不同的项目参与者都可以对项目进行修改并commit版本，比如可以清晰的看到项目整个开发过程。
举个栗子：

版本 用户 说明 日期
1 张三 删除了软件服务条款5 7/12 10:38
2 张三 增加了License人数限制 7/12 18:09
3 李四 财务部门调整了合同金额 7/13 9:51
4 张三 延长了免费升级周期 7/14 15:17

整个项目是不是很清楚明了？

2.什么是Coding？Github？

  百度百科定义：GitHub 是一个面向开源及私有软件项目的托管平台，因为只支持 Git 作为唯一的版本库格式进行托管，故名 GitHub。
  这里就介绍github毕竟太出名了,世界各地的大牛都在这上面托管项目；而coding是国内的，语言中文就方便了不少，而且反应速度更快。就像上面讲的，git进行版本控制，但要实现分布式，你得有一个网络平台，而Github就提供了这样条件。
  我们在本地实现了一个好玩的牛叉的项目，没同行看到岂不是锦衣夜行（这仅仅是比较low的想法），所以托管到Github上让其他人也来观摩观摩，让其他也参与这个项目中，如果有意见有好点子，也是可以加到我的代码的。
  具体的大家就看看关于Github的百度百科吧。

git的基本操作

这里要介绍下git的工作原理，git有三个区，姑且称之为：工作区，暂停区和历史提交区，此外还有远程库。
首先我们在项目所在文件夹git init建立一个隐藏文件.git也就是创建工作区。然后git add <files>将选择的文件放入暂停区，通过git status可以查看放入暂停区的文件，通过文件颜色可以看到本次提交的文件和上次提交版本有什么区别。git commit正式提交后会出现vim界面，让你输入本次提交的注释，也可以git commit　-m "XXXXX"讲注释放入“”中。最后git log可以查看历次的版本提交。
